# DreamHome 资产生产线技术解析(路演 / 技术报告素材)

> 一句话:把任意一条抖音家装视频,全自动变成「可暂停识别 + 可 3D 查看」的家具资产库。
> 以真实案例贯穿:56平两室一厅一镜到底视频,248 秒。

## 流程总览

```
抖音链接 → ①下载抽帧 → ②开集检测 → ③碎轨迹拼接 → ④SAM2全帧追踪
        → ⑤外观聚类 → 人工预审(5分钟) → ⑥五道质量闸+3D生成 → ⑦资产库+时空索引
```

## 各阶段详解(带真实数字)

### ① 下载 + 抽帧
248 秒视频,0.5s 网格抽帧 → **498 张关键帧**。

### ② 开集检测(Grounding DINO,自部署 GPU)
逐帧找家具:框 + 品类(沙发/床/柜子/卫浴/家电…共 20+ 词)→ **1479 个检测框**。
- 开集检测 = 词表可随时扩充,不需要重训模型(比赛期间加过"马桶、电视柜、衣柜"等)
- 工程优化:词表按 ≤5 词分组防 prompt 稀释、同图多 prompt 单批前向、跨标签 NMS,
  单帧 5.3s → **0.35s**(15 倍提速)

### ③ 碎轨迹拼接(IoU 关联)
相邻帧重叠框串联 → **193 条碎轨迹**。
镜头扫动/遮挡会把同一件家具打碎成多段 —— 193 是"原材料",不是家具数。

### ④ SAM2 全帧追踪(Meta SAM2,自部署,T1C1 方案)
193 条碎轨迹各取最清晰一帧做"种子",SAM2 在整条视频上传播出**像素级轮廓轨迹**:
1. **定位准**:暂停视频时光点位置逐帧精确(旧方案靠稀疏检测框插值,会漂)
2. **碎片归一**:同帧轮廓高度重叠(IoU≥0.6)的种子自动合并 → 193 段收敛到几十件
3. **外观指纹去背景**:用轮廓抠图(白底)后再算 CLIP 向量,多帧平均 ——
   解决了"两个不同吊灯因为背景相似被认成同一个"的经典错误
- 显存工程:与 TRELLIS 共卡 22GB,帧张量 offload 到内存、种子分批(6/批)传播

### ⑤ 外观聚类 + 人工预审
按外观指纹归堆(同品类阈值分级:小物件更严),产出**聚类预审页**:
每件家具一行,列出它的全部片段截图。人工 5 分钟扫一遍,修正记成 JSON
(合并/拆分/丢弃),重跑时生效 —— **在花钱生成之前把关废品**。

### ⑥ 五道质量闸 + 3D 生成(不合格不花钱)
每件依次过:
1. 尺寸/亮度闸(太小太暗 → 跳过)
2. 切边闸(被画面裁掉 ≥2 边 → 跳过,防 3D 脑补形态)
3. 品类闸(qwen-vl 看原图定品类,检测误分类在此纠正;"其他" → 打回)
4. **补全分流**(降本核心):qwen-vl 同一次调用顺带判断"是否完整可见" ——
   完整 → 直接进 3D(¥0);有遮挡 → wan2.7-image 补全成产品图(¥0.2,
   品类约束防"床补成椅子")。A/B 实测 ¥0.2 模型与 ¥0.5 pro 质量打平
5. 幻觉闸(补全图必须还是原图那件家具,不一致 → 打回;真实拦截过
   "厨房水槽被补全成陶瓷容器")
→ TRELLIS(自部署,fp16,A10 实测 38s/件)生成 GLB → 材质修正(trimesh
导出默认发黑的坑)→ 自动打标签(品类/颜色/材质/风格)→ 入库

### ⑦ 产物
- **资产库**:3D 模型 + 结构化标签 + 外观向量(驱动"疑似重复"合并)
- **时空索引**:每件家具在视频每 0.2s 的位置 → App"暂停出光点、点开看 3D"
- 演示页:播放原视频 → 暂停 → 白点呼吸标记 + 名称 → 点开 3D 查看

## 关键工程决策(技术报告可展开)

| 决策 | 理由 |
|---|---|
| 全自部署(阿里云 A10 单卡)| fal.ai 单张 $0.1+,自部署回本点极低;比赛在国内,链路全直连 |
| 内容寻址缓存(检测/追踪/向量/补全/生成全环节)| 中断重跑零成本;改参数只重算变化环节;预审页与真实聚类同源 |
| 试产 3 件 → 人工确认 → 放全量 | 质量问题在 3 件内暴露,不浪费批量成本 |
| 宁拆勿合的聚类阈值 | 拆错可在预审页一键合并;合错(两件家具混一簇)不可修 |
| 失败必须大声 | worker 自愈(CUDA 脏状态自杀重启)、超时必报、GPU 任务队列可查 |

## 成本(单条视频,~30 件资产)
- GPU:按量 A10,生产窗口制,¥15-25/天
- API:补全分流后 ~¥3/视频(原 ¥15) + 打标签/质检 ~¥0.3
- **单件资产综合成本 ≈ ¥0.1-0.2**,对比 fal.ai 单件 ¥0.7+

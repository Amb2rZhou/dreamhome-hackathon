# 轨迹手动矫正 SOP

给人工矫正"暂停识别游标指错/飘走"用的工具流程。配套页面 `/review/fix.html`，
后端端点在 `backend/app/routers/tracks_fix.py`（`/api/tracks/{track_id}/unbind|rebind|cut`）。

## 打开页面

1. 确认 backend 在跑：`curl http://127.0.0.1:8000/api/health`
2. 浏览器打开 **http://127.0.0.1:8000/review/fix.html**（可带 `?video_id=vid_xxx` 直达）
3. 顶部下拉选视频 → 播放 → 在出错的时刻**暂停** → 画面上出现游标：
   - **实心白点** = 该轨迹已绑定资产（标签是资产名）
   - **粉色空心点** = 未绑定轨迹（标签是"品类·未绑定"）
   - 勾选"同屏去重"可开关重叠游标合并；勾选"显示巨框"能看到默认隐藏的半屏级大框轨迹
4. 点击游标 → 右侧栏显示该轨迹信息（track_id、品类、时间段、帧数）和当前绑定资产缩略图

## 四种错误类型 → 对应操作

| 错误现象 | 用哪个操作 | 效果 |
|---|---|---|
| 游标位置对，但**指错了物体**（比如沙发的点标成了茶几） | **换绑** | 点"换绑…"展开本视频资产列表（缩略图+名字），点正确的那个即可改绑 |
| 轨迹大体正确，但**某一小段光点飘走**（跟丢/漂移） | **解绑当前时段** | 播放到飘走处暂停，默认剥离当前时刻前后各 2s（可改秒数）。该时段的采样点从原轨迹删除并另存为一条未绑定新轨迹；原轨迹其余部分保持绑定 |
| **整条轨迹都是错的**（从头到尾都不是这个物体） | **整条解绑** | asset_id 置空，轨迹几何保留（变粉色空心点），之后可再换绑或不管它 |
| **一条轨迹先后跨了两个物体**（镜头切换后跟到了别的东西） | **切断** | 播放到两个物体交界处暂停，点切断。前段保留原 track_id 和绑定；后段变成新的未绑定轨迹，再点后段游标单独"换绑"到正确资产 |

边界行为：
- "解绑当前时段"若剥离后剩余采样点不足 2 个，自动退化为整条解绑（toast 会提示）。
- 被剥离的时段若不足 2 个点，不会另建新轨迹（点太少没有保留价值）。
- "切断"要求切点两侧都有采样点，切在轨迹范围外会报错。

每次操作成功后页面自动重拉索引、重画游标，底部 toast 显示结果（含新轨迹 id）。

## 改完复验

1. 打开 **http://127.0.0.1:8000/review/video.html?video_id=vid_xxx**（正式演示页）
2. 在刚才矫正过的时刻暂停，确认：
   - 换绑后的游标标签显示正确的资产名，点开侧栏 3D 模型是对的物体
   - 解绑过的时段不再出现错误光点（video.html 只显示已绑定且 ready 的资产）
   - 切断后的前段仍正常，后段换绑后也正确
3. 需要看全部轨迹（含未绑定）时用 `video.html?all=1` 预审模式对照。

## 注意

- 该工具只改 `tracks` 表（asset_id / frames_json / 时间字段），不动 `assets` 表，
  与量产批次进程并行安全。
- 若刚重启过 backend 或端点 404：backend 不是 `--reload` 模式时，新端点需要重启
  `uvicorn app.main:app --port 8000` 才生效。

## 1. 产品定位

DreamHome 是一本**家的灵感摘抄本**：把用户在抖音刷到、线下看到、脑子里想到的"好看"，通过视觉+语音摘抄成带信息的 3D 组件，最终在自己的户型里组装成"梦想中的家"，并作为一个可被好友访问的社交空间被分享出去。

赛道契合点：**一次看见，自然延展为理解与互动**——看见（视觉识别）→ 摘抄（理解成结构化组件）→ 组装（语音互动操控）→ 邀请进入（社交延展）。

核心体验闭环：
```
看见（视频圈选 / 线下拍照 / 手绘）
   ↓ 摘抄（SAM 抠图 → 去打造 3D）
灵感素材库（每个元素都是带信息的 3D 组件）
   ↓ 组装
扫描户型图 → 生成自己家的 3D 空间 → 把摘抄的元素摆进去
   ↓ 分享
邀请抖音好友来"梦想中的家"里玩
```

## 2. 当前 demo 现状（已实现功能）

> 本节回答"现在 demo 做到什么地步"。纯前端 React+Vite 原型，3D 用 three.js 程序化生成（零外部模型资源），后端抠图用 rembg（见 §6）。

### 2.1 视频圈选摘抄（feed）✅
仿抖音手机壳页面，真视频 + canvas 画笔。
- **浏览态（browse）**：全屏播放 `public/videos/home-1.mp4`，抖音式社交栏（头像/关注/点赞/评论/收藏/分享/音乐盘）+ 底部作者信息。点屏幕进入摘抄态。
- **摘抄态（session）**：画笔圈选（canvas 贝塞尔曲线笔触 + 笔尖跟随 + 边界框计算）+ "识别全部"全帧检测。识别后浮卡叠加在视频上，支持点选/删除单个，底部"已摘 N"+"↓ 存入"。
- **SAM 真实抠图**：圈选/识别全部后，截帧 + bbox POST 到后端 `localhost:8001/api/segment`（rembg 去背景），返回透明 PNG 替换 canvas clip 占位。先出 clip 占位（即时反馈），异步替换为干净贴纸。失败保留 clip。
- **删除按钮边界**：obj-card 的 × 按圈选框位置动态贴角（贴右屏边翻左、贴顶翻下），保证始终在屏内可见。
- **AI 输入框**：底部文本输入框（"可以说：帮我圈选台灯"），语音入口的视觉占位。

### 2.2 2D→3D 打造流程 + 全局桌宠 ✅（本次新增）
摘抄后的 2D 贴纸 → 3D 组件的生成等待态，用全局桌宠"包公球"承载。
- **入口（preview）**：摘抄存入前弹 preview，文案"识别成功啦，去打造它们的 3D 形态吧（约30~120秒）"。底部"素材库"（灰）+ **"去打造 →"**（黑）统一按钮，圈选的多件**作为一批**一并去打造。
- **接单加载弹窗**：点"去打造" → 独立弹窗（portal 到 body，全屏居中）：包公球左右踱步（thinking 态，转身动画）+ "包公球接单中…" + "正在确认 N 件家具的打造任务" + 进度条，**6s** 前端过渡（模拟后端接单/排队接口）。
- **开工提示气泡**：弹窗结束 → 桌宠切 working 焊接态 + 弹一次性气泡"开工啦！大概需要 2 分钟，你可以继续浏览，好了我告诉你"（3.5s 自动消失或 × 关）。气泡位置自适应：桌宠靠右时气泡向左展开、靠左时向右，不出屏。
- **桌宠焊接态**：全局悬浮桌宠（右下角，可拖动）working 形态（敲打晃动 + 火花动效）。拖动需按住移动 >5px 才触发（单击不拖）。约 4.5s/件（演示压缩，真实 30~120s）。
- **按批聚合完成**：一批多件单线程依次焊完，**全批完成后**桌宠变 happy + 气泡"✨ 本批 N 件都打造完啦！"（单件则"XX打造完啦"）。点"看看"看本批全部成果。
- **半页成果浮层（多件）**：点"看看" → 半页浮层：上方 3D 预览（可拖转，松手自动转）+ 下方缩略图横滑切换不同件 + 当前件信息 + **"一键存进仓库（N 件）"**按钮，无别的出口。退出（✕/遮罩）→ 二次确认"还没存进仓库/放弃本批 N 件"。
- **单线程 + 多批**：打造中再点"去打造" → 吐司"包公球还在打造上一批，新的一批已排队"，新批入队，上一批完成后自动接上。
- **桌宠平时态**：sleeping（呼吸）/ happy 蹦跳，全局悬浮不挡浏览，可拖到任意位置。
- 6 状态图：initial/idle/thinking/working/happy/sleeping，存 `public/mascot-*.png`。当前用 sleeping/happy/working/thinking 四态。
- **多批任务待建**：当前一批聚合展示已做。多批（跨批）的队列管理、跨批气泡/视图同步、与素材库/组装页状态联动尚未做（见 §8.2）。

### 2.3 素材库 ✅
平铺网格（不分类），2D 抠图贴纸形态。
- **贴纸形态**：卡片用 2D 抠图贴纸（非 emoji）。seed 数据用 `stickerGen.ts` 程序化生成 6 品类 SVG 剪影（沙发/茶几/吊灯/绿植/装饰画/地毯，品类色渐变 + drop-shadow + 轻微旋转）；摘抄/打造来的用真实 snapshot。
- **详情浮层**：点卡片弹 sheet，three.js 3D 预览（可拖转、松手自动转）+ 来源/尺寸/风格标签 + 删除。
- **FAB 新增入口**：右下角 + 展开三渠道——视频圈选（回 feed）/ 线下拍照（mock 加组件）/ 手绘生成（mock 加组件）。mock 加的标"预生成"。
- **新家具入场动画**：保存同款家居/打造存入后，新组件逐个入场动画（从上方落下+回弹，错开 0.06s）+ 左上角青色"新增"角标，4s 后消失。
- **去组装出口**：右上角"去组装 →"。

### 2.4 博主同款小家 ✅（本次新增）
feed 作者区 + 博主主页 + 半页浮层获取。
- **feed 作者区卡片**：@作者上方，灰色半透明圆角方框（毛玻璃），上下布局——上文"获取博主同款小家"+副标题"含奶油风一居小家，6件软装"，下 3D 微缩预览（fillContainer 撑满）。卡片 inline-flex 自适应宽度，靠左不挤社交栏。
- **3D 微缩动效**：转一下停（转 1.4s 升至俯视、停 2.6s 循环），定格俯视全览。
- **半页浮层（点卡片）**：从底 scaleY 展开（和摘抄页 preview 一致动效）。结构：顶部固定（标题）+ 3D 预览固定 200px（定格俯视，可拖转）+ 家具网格独立滚动 + 底部两按钮"保存同款家居 / 保存整个小家"。
  - 保存整个小家 → 设 layout 为博主户型 + 组件加库 + 进组装页，吐司"已保存整个小家，去组装看看"
  - 保存同款家居 → 6 件单品存库 + 进素材库，吐司"已保存至素材库"，新组件带入场动画+新增标记
- **博主主页（点 @作者）**：抖音作者首页布局，"进入店铺"位换成"进入我的小家"卡片（3D 预览 + 进入）。头像/名/简介/粉丝/获赞/小家组件数，关注+私信，小家 tab，组件网格点开详情浮层。
- 数据：`CURRENT_BLOGGER`（家居灵感研究所）+ `BLOGGER_HOME_PACK`（6 件奶油风组件）+ `DEFAULT_LAYOUTS` 里的一居一厅户型。

### 2.5 3D 组装编辑器 ✅
three.js 3D 空间 + 户型选择 + 拖放摆放 + 语音占位。
- **户型选择入口**：首次进组装页弹 LayoutPicker（首次必选）：
  - 默认户型 3 个：开间 / 一居一厅 / L 型（带缩略图）
  - 上传户型图：拍/相册选 → "解析中"进度（2s mock）→ 预设房间
  - 自定义画墙：6×6 网格点选连成多边形（≥3 点自动闭合）→ 生成 3D
- **3D 空间**：按户型生成地板 + 墙体 + 网格，自然光 + 阴影。
- **相机操控**：空白处拖转视角、滚轮缩放。
- **摆放**：底部组件面板（素材库横向滑动），点一下生成 3D 家具落到房间中央环形分布。拖家具可移动（限制在房间内）。
- **选中操作条**：点 3D 物体选中，底部黑条 → 拖动移动 / 🎤"挪到窗边"（语音占位，挪到预设位）/ 删除。
- **换户型**：顶部户型标签可点重选。


## 6. 后端抠图服务（rembg）

**状态**：代码就绪，依赖装好，服务能起，**卡在 u2net 模型下载**（176MB，github ~50kB/s，huggingface 超时）。模型未到位前，前端抠图失败兜底用 canvas clip 占位。

- 服务：`backend/segment_api.py`，FastAPI，端口 **8001**（8000 被其他服务占）
- 接口：`POST /api/segment`（file + bbox=x,y,w,h）→ 裁剪 + rembg 去背景 → 透明 PNG
- 启动：`cd backend && .venv/bin/uvicorn segment_api:app --host 127.0.0.1 --port 8001`
- 模型路径：`~/.u2net/u2net.onnx`（rembg 首次运行自动下，需手动下或换源）
- 依赖：`rembg[cpu]`（带 onnxruntime）、pillow、fastapi、uvicorn、python-multipart

**升级路径**：接口写成可替换的，未来换真 SAM（promptable，框提示抗"圈选不准"）只需改后端实现，前端不动。SAM ViT-B onnx 推理峰值 ~1.2-1.8GB，当前机器 16GB 偏紧，暂用 rembg（~500MB）。

## 7. 关键技术决策记录

| 决策 | 选择 | 理由 |
|---|---|---|
| 抠图方案 | rembg + 框内去背景 | 内存安全（~500MB vs SAM 1.8GB），演示稳，留 SAM 升级口 |
| 3D 资源 | three.js 程序化生成 | 零外部模型，演示稳，6 品类简化几何体 |
| 贴纸形态 | 程序化 SVG 剪影（mock）+ 真实 snapshot（摘抄） | 统一"2D 抠图贴纸"视觉，不用 emoji |
| 2D→3D 入口 | preview 内"去打造"按钮 | 摘抄当下决定，一次性心智 |
| 打造期间 | 方案甲：后台 + 全局桌宠 | 不影响浏览，桌宠跨页通知 |
| 打造并发 | 单线程排队 | 桌宠只有一个，物理合理 |
| 接单态 | 纯前端过渡 1.6s | 模拟后端接单/排队接口，演示稳 |
| 半页成果出口 | 只能存，退出二次确认 | 强制预览确认，避免误弃 |
| 博主入口 | feed 作者区卡片 + 主页并列 | 单件摘抄 + 整包获取双形态 |
| 户型选择 | 首次进组装必选 | 符合"开始装扮自己的家"心智 |

### 8.3 演示策略重大转向（2026-07-13）

**原方向**：feed 实时识别展示（刷视频→实时圈选→实时 SAM→实时 3D），全链路现场跑。
**根本问题**：① 后端难做、时延太长；② **AI 产品不稳定性强**，现场全链路跑风险太高。要做"安全方案"。
**新方向（安全方案）**：
- **主舞台 = 素材库 → 组装**：素材库作为"已识别产物"直接展示（离线预置好），组装是演示核心。把时延和 AI 不稳定藏到幕后。
- **feed 仍要做，目标"跑通"非"调优到极致"**：演"摘抄"概念（圈选→SAM 抠图出 2D 贴纸，秒级），不演 3D 生成等待。SAM 照样做。
- **每个素材可回溯原视频**：组件加溯源属性（sourceVideo + sourceFrame），素材库详情 + 组装页选中都露出，点跳回 feed 那帧。用溯源代替实时，既避时延/不稳定，又保可信度 + 情感（"我的家是我刷过的所有好看拼成的"）。
**本质**：用"溯源"换"实时"——绕开 AI 不稳定，增强可信度与情感。演示展示体验，不展示工程难度。
**影响动线**：演示主线 = 素材库（已有）→ 组装；feed = 概念演示 + 摘抄入口（跑通即可）。
